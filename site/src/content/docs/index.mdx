<CardGrid>
  <Card title="Agent Autonomy" icon="rocket">
    Runners are autonomous — configured once at startup, they own their session state, prompt, tools, and workspace. The router is a thin dispatcher.
  </Card>

  <Card title="Model-Agnostic" icon="puzzle">
    Claude Code, Ollama, OpenAI-compatible, or Gemini. Pick the LLM engine per agent. Downgrade automatically when budgets run low.
  </Card>

  <Card title="Secure by Default" icon="approve-check-circle">
    Deny-by-default permissions. Agents get zero tools unless explicitly allowed. Budget enforcement blocks runaway spending.
  </Card>

  <Card title="9 Channels" icon="comment-alt">
    Telegram, Slack, WhatsApp, Signal, Discord, Webhook, iMessage, Google Chat, and Web Chat. One config, every frontend.
  </Card>

  <Card title="Semantic Memory" icon="open-book">
    SQLite-vec powered memory with hybrid search, MMR reranking, temporal decay, and automatic prompt injection.
  </Card>

  <Card title="Tool Registry" icon="puzzle">
    Governed tool execution with built-in web fetch, search, PDF parsing, and image analysis. Plugins register custom tools.
  </Card>

  <Card title="Docker Isolation" icon="seti:docker">
    Sandbox by default — Docker containers with memory/CPU limits, private workspaces, and browser sidecars.
  </Card>

  <Card title="Voice + Canvas" icon="laptop">
    TTS/STT with 6 providers. A2UI canvas protocol pushes forms, tables, and charts to companion UIs over WebSocket.
  </Card>

  <Card title="Automation" icon="star">
    Cron scheduling and webhook receivers for autonomous agent work. Job history with idempotency dedup.
  </Card>

  <Card title="23-Check Security Audit" icon="approve-check-circle">
    Continuous auditing: permissions, budgets, channels, memory, browser, automation, gateway, governance, and runtime anomalies.
  </Card>

  <Card title="Plugins + Skills" icon="document">
    Plugin registry with manifests, lifecycle management, and SKILL.md auto-discovery for system prompt injection.
  </Card>

  <Card title="Federation (BR Platform)" icon="random">
    Governed agent bus with two-layer auth. Federation routes calls across nodes. BR platform for enterprise management.
  </Card>
</CardGrid>
